# 코테 문제 풀이 프로세스

## 1) 문제를 읽을 때 체크할 것
- 입력 크기 `N` 확인
- 출력 조건과 예외 조건 확인
- 여러 테스트 케이스 여부 확인

## 2) 접근 방식 선택 기준
- `N <= 20`: 완전탐색 가능성 먼저 확인
- `N <= 1,000`: O(N^2) 가능 여부 확인
- `N <= 100,000`: O(N log N) 이하 우선
- 그래프/연결: DFS/BFS 우선
- 최솟값/최대값 누적 최적화: DP/그리디 검토

## 3) 풀이 작성 순서
1. 입출력 뼈대 작성
2. 핵심 자료구조 정의
3. 작은 예제로 손으로 시뮬레이션
4. 본 코드 작성
5. 경계값 테스트 (`1`, `0`, 최대치)

## 4) 디버깅 방식
- 실패 시 바로 전체 수정하지 말고, 반례 1개 고정
- 출력 중간값(`print`)으로 상태 추적
- 인덱스/초기값/반복문 범위부터 점검

## 5) 복기(실력 상승 핵심)
- 틀린 이유 1줄 기록
- 정답 아이디어 3줄 요약
- 다음날 같은 문제 다시 풀이
